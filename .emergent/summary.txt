<product_requirements>
The primary objective is to build and stabilize a data-consistent, engaging real-time quiz application with social features.

**Core Functional Requirements:**
1.  **Single Source of Truth:** All application data (exams, questions, user profiles, social interactions) must originate from a single, consistent source in the MongoDB database, eliminating hardcoded or conflicting data structures.
2.  **Social Networking:** Users must be able to follow/unfollow each other, view user profiles, and engage with a social feed. The system must accurately track and display follower/following counts and lists. Follow requests to private profiles must require approval.
3.  **Content Creation & Interaction:** Users must be able to create posts on a social feed, which should appear in real-time at the top of the feed for other users.
4.  **Stable Navigation:** The application must adhere to standard web navigation patterns. Users should not be redirected unexpectedly when refreshing the page or using the browser's back button.

**UI/UX Requirements:**
1.  **Conditional Views:** The UI must intelligently display different views based on user context. For example, a user clicking their own profile/post should be directed to an editable dashboard, while clicking on another user's content should lead to a public-facing profile.
2.  **Modern UI Elements:** The application should use modern, consistent UI elements, such as SVG icons instead of text-based emojis, for actions like 'like' and 'comment'. Emojis used for reactions should render correctly across all platforms.
3.  **New Content Integration**: The system must support the addition of new exam structures, as demonstrated by the request to add 10 new banking exams from a PDF.
</product_requirements>
